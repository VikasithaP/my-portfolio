1. Personal Portfolio Repository
 
Build a small HTML/CSS/JS portfolio and host it with GitHub Pages.
 
What you practice:
 
creating a repository

commits and commit history

README writing

branches for feature changes

pull requests

GitHub Pages deployment
 
Suggested tasks:
 
create index.html, style.css, script.js

make separate branches for navbar, about section, contact form

merge through pull requests

publish the site using GitHub Pages
 





2.Team Documentation Project
 
Create a documentation repo for a topic such as Linux commands, SQL basics, or cloud notes.
 
What you practice:
 
markdown

issues

pull requests

branch-based editing

review workflow

wiki or docs structure
 
Suggested tasks:
 
create folders like basics, advanced, examples

open issues for missing topics

create a branch per topic

submit pull requests with changes

review and merge updates









3. Open Source Contribution Simulator
 
Create your own “main repo” and then fork it using another GitHub account or simulate fork workflow locally.
 
What you practice:
 
forks
upstream and origin
pull requests from fork to parent repo
syncing fork with main repository
contribution etiquette
 
Suggested tasks:
 
create an original repo called python-utils
fork it
add a feature in the fork
raise a pull request back to the original repo
make upstream changes and sync the fork
